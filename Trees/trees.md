Trees (Specifically Heaps & B-Trees)
Applications that use trees are: Database Indexing (MySQL, PostgreSQL).

Why: Databases use B-Trees because they allow for searching, sequential access, insertions, and deletions in logarithmic time O(logn), which is vital when dealing with millions of rows.

Algorithm Example: Heap Sort. Used in priority queues where the "most important" task (like a system interrupt) must always stay at the top.
